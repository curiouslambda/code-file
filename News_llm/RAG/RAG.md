# Retrieval-Augmented Generation (RAG)

## 1. RAG란?
RAG(Retrieval-Augmented Generation)는 기존의 생성형 AI(Generative AI) 모델에 정보 검색(Retrieval) 기능을 추가하여 더 정확하고 신뢰할 수 있는 응답을 생성하는 방식이다. RAG는 크게 Retrieval(검색)과 Generation(생성) 두 부분으로 나뉘며 필요할 때 외부 데이터를 검색하여 이를 참고한 후 응답을 생성한다.

## 2. RAG의 주요 개념

### 2.1 검색 (Retrieval)
- 미리 구축된 벡터 데이터베이스(예: FAISS, Chroma, Pinecone, 사내문서서 등)에서 관련 정보를 검색함
- 질문(Query)과 문서(Document)를 벡터 임베딩(Embedding) 방식으로 변환하여 유사도가 높은 문서를 찾아냄
- 검색된 문서를 바탕으로 생성 모델이 답을 생성하는데 활용

### 2.2 생성 (Generation)
- 검색된 문서와 함께 입력 쿼리를 활용하여 생성 모델이 응답을 생성
- GPT, BART, T5 등의 사전 학습된 언어 모델을 기반으로 동작합니다.
- 단순한 텍스트 완성이 아닌, 검색된 정보와 기존 지식을 결합하여 답변을 강화함으로써 정확도를 높임임

## 3. RAG의 아키텍처
RAG 모델은 **RAG-Sequence**와 **RAG-Token** 두 가지 방식으로 나누어짐짐

### 3.1 RAG-Sequence
- 검색된 문서 중 하나를 선택하여 이를 기반으로 문장을 생성하는 방식
- 문서 간의 일관성을 고려하지 않지만 빠르게 응답을 생성할 수 있음음

### 3.2 RAG-Token
- 생성 과정에서 각 토큰을 예측할 때마다 검색된 문서 집합을 활용하는 방식
- 보다 정확한 응답을 생성할 수 있지만 연산 비용이 큼큼

## 4. RAG의 장점
✅ **외부 지식 활용**
- 사전 학습 데이터에 포함되지 않은 최신 정보도 반영 가능 <br>
  
✅ **정확성 향상**
- LLM이 훈련 데이터만으로 답을 생성하는 것이 아니라 실시간으로 검색한 최신 정보를 활용하기 때문에 신뢰할 수 있는 답변 생성 <br>
- 잘못된 정보(할루시네이션, Hallucination)를 줄이는 데 도움을 줌 <br>

✅ **확장성**
- 도메인별 데이터베이스를 활용하여 특정 분야에 특화된 응답 가능 <br>
- 최신 연구 논문, 기업 내부 문서, 뉴스 등을 실시간으로 참조 가능 <br>


## 5. RAG의 한계
❌ **검색 의존성**
- 검색된 문서가 부정확할 경우, 생성 결과의 품질이 저하될 수 있음 <br>

❌ **추론 속도 저하**
- 검색 및 생성 과정을 포함하므로 응답 속도가 느릴 수 있음 <br>

❌ **지식 결합 문제**
- 검색된 문서와 언어 모델의 사전 지식이 충돌할 가능성 있음 <br>

## 6. RAG의 활용 사례
- **QA 시스템**: 고객 지원 챗봇, 법률/의료 분야의 질문 응답 시스템
- **문서 요약**: 대량의 문서에서 핵심 정보 추출 및 요약
- **코드 자동 생성**: 프로그래밍 질문에 대한 코드 추천
- **학술 연구 지원**: 논문 검색 및 관련 연구 요약
