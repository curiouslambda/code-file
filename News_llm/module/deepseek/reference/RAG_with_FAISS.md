✅ RAG + FAISS의 정확한 흐름
원본 텍스트 → 벡터화 → FAISS에 저장 (사전 구축)

문서(텍스트)를 벡터로 변환 (임베딩 모델 사용)
FAISS 인덱스에 저장 (빠른 검색을 위해)
LLM에 사용자 질문 입력

사용자가 질문을 입력
질문을 벡터화 → FAISS에서 유사 문서 검색

질문을 같은 임베딩 모델을 사용해 벡터로 변환
FAISS에서 가장 유사한 문서 검색 (k-NN 검색)
검색된 문서를 기반으로 프롬프트 생성

FAISS에서 검색된 문서(원본 텍스트)를 활용하여 LLM 프롬프트 구성
LLM이 프롬프트를 바탕으로 답변 생성

검색된 문서를 참고하여 답변을 생성

(1) 사전 구축 단계

원본 문서 → 벡터 변환 → FAISS에 저장
(2) 질의 응답 단계

사용자의 질문 → 벡터 변환 → FAISS에서 관련 문서 검색 → 검색된 문서로 프롬프트 구성 → LLM이 답변 생성
