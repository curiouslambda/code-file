# 태그 데이터 수집

## 작동 순서
- 데이터 유입
- 프롬프트 선택 - 현재 작업에 맞는 여러 하드코딩된 프롬프트 중 선택(사용자 입력)
- 프롬프트 개선(RAG) - FAISS를 사용하여 입력값과 가장 비슷한 문서를 찾아서 프롬프트 개선
- Answer
- 답변 검증
- 답변 저장
- 답변 벡터화 후 RAG 데이터 업데이트

## 데이터 구성
1. RAG데이터 
   1. 수집한 데이터 벡터화
   2. FAISS를 사용하여 데이터 저장 
2. 프롬프트 - 답변 데이터

# 태그 주식 연결

## 작동 순서
- 태그, 기간 입력
- 태그가 그래프가 있는 경우 그래프를 출력하고 수집한 모든 태그가 포함되어 있는 뉴스 리스트 출력
- 하락하는 기간, 상승하는 기간을 시각적으로 보여주고 찾는다

## 데이터 수집
- 태그 입력시 오늘 기준 데이터가 있는지 확인하고 존재하지 않을 시 yahoofinance api 요청함
- 저장된 태그 - 뉴스 데이터에서 태그가 존재하는 기간 동안의 모든 뉴스 시간 순으로 정렬 후 출력
