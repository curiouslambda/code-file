# News_llm

뉴스 데이터를 수집하고 전처리한 뒤, LLM과 RAG를 활용해 태그 추출과 질의응답 흐름을 실험한 프로젝트입니다.

## Project Overview

이 프로젝트는 뉴스 텍스트를 단순히 수집하는 데서 끝나지 않고,  
크롤링된 데이터를 구조적으로 다루고, 임베딩 기반 검색과 LLM 응답 생성까지 연결하는 흐름을 정리하기 위해 진행했습니다.

특히 뉴스 데이터를 기반으로

- 필요한 본문을 수집하고 정리하는 단계
- 프롬프트와 응답 흐름을 관리하는 단계
- 벡터 검색을 통해 관련 문서를 찾아 LLM 응답을 보강하는 단계
- 태그와 뉴스 데이터를 이후 분석에 활용할 수 있도록 저장하는 단계

를 하나의 파이프라인 관점에서 다뤄보는 데 초점을 두었습니다.

## Goal

- 뉴스 데이터 크롤링 및 본문 추출
- 전처리와 프롬프트 관리 구조 정리
- LLM 기반 태그 추출 및 응답 생성 실험
- FAISS 기반 검색을 활용한 RAG 흐름 구성
- 태그와 뉴스 데이터를 구조적으로 저장할 수 있는 기반 마련

## Why This Project Matters

단순히 "LLM을 붙여본 것"이 아니라, 뉴스 데이터 수집부터 검색 기반 응답 생성까지 이어지는 실제 파이프라인을 설계하고 정리해본 경험입니다.

또한 한국어 뉴스 데이터를 다루는 과정에서

- 크롤링 결과를 어떻게 문서 단위로 나눌지
- 어떤 프롬프트 구조로 태그를 추출할지
- 검색 결과를 어떻게 응답 생성에 연결할지
- 한국어와 영어가 섞이는 모델 입출력을 어떻게 다룰지

같은 실제적인 문제를 함께 고려했다는 점에서 의미가 있습니다.

## Main Work

- 특정 URL에서 뉴스 HTML을 수집하고 본문 텍스트를 추출하는 크롤링 흐름 정리
- 크롤링 결과를 후속 처리에 사용할 수 있는 형태로 정리
- 프롬프트 템플릿 및 프롬프트 관리 구조 검토
- 임베딩과 FAISS를 활용한 검색 기반 RAG 실험
- 로컬 LLM 호출 모듈과 번역 모듈을 연결한 질의응답 흐름 정리
- 태그 추출 및 저장 구조를 문서화하고 확장 방향 설계

## Pipeline

1. 뉴스 URL 또는 대상 데이터를 입력으로 받습니다.
2. 크롤러가 HTML을 수집하고 본문 텍스트를 추출합니다.
3. 추출된 텍스트를 문서 단위로 정리하고 후속 처리에 사용합니다.
4. 임베딩을 생성하고 FAISS 인덱스를 통해 관련 문서를 검색합니다.
5. 검색 결과를 바탕으로 LLM이 태그 추출 또는 질의응답을 수행합니다.
6. 필요 시 번역 모듈을 통해 한국어/영어 입력 흐름을 보정합니다.
7. 결과와 로그를 저장해 이후 재사용 또는 확장을 고려합니다.

## Directory Structure

- `module/crawler/`
  - 뉴스 HTML 수집 및 본문 추출 관련 코드
- `module/deepseek/`
  - LLM 호출, 번역, RAG 실험, 프롬프트 및 로그 관련 코드
- `RAG/`
  - RAG 개념 정리 및 실험 자료
- `temp/`
  - 임시 실험 노트북
- `ISSUE.md`
  - 우선순위 및 진행 중 항목 정리
- `PROJECT_OVERVIEW.md`
  - 내부 구조와 주요 스크립트 중심의 기술 개요 문서
